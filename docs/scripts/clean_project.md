# Скрипт: `clean_project.sh`

Скрипт предназначен для очистки мусора в Python-проекте.  
Удаляет временные файлы, кэш и артефакты сборки, чтобы поддерживать проект в чистоте.

---

## Использование

```bash
# Удалить весь мусор
./scripts/clean_project.sh

# Dry-run: показать, что будет удалено, но не удалять файлы
./scripts/clean_project.sh -n
```

**Параметры:**

* `-n` или `--dry-run` — режим показа, скрипт выводит, что будет удалено, но не удаляет файлы.

---

## Что удаляет

Скрипт ищет и удаляет следующие типы файлов и папок:

* `__pycache__` — кэш компиляции Python
* `.mypy_cache` — кэш `mypy`
* `.pytest_cache` — кэш `pytest`
* `.ruff_cache` — кэш `ruff`
* `.cache` — общий кэш
* `build/`, `dist/` — артефакты сборки
* `*.egg-info`, `pip-wheel-metadata` — артефакты установки пакетов
* `.venv`, `venv` — виртуальные окружения
* `.ipynb_checkpoints` — контрольные точки Jupyter
* `.DS_Store` — файлы macOS
* `.coverage`, `htmlcov` — файлы покрытия тестов
* `.tox`, `.eggs` — артефакты тестирования и пакетов
* `*.pyc` — скомпилированные Python-файлы
* `.benchmarks` — данные для бенчмарков

**Примечание:**
Файлы `.env` не удаляются даже при совпадении с паттерном.

---

## Реализация

Скрипт использует следующие конструкции:

* `set -euo pipefail` — строгий режим для обработки ошибок
* Определение корня проекта:

  ```bash
  ROOT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )/.." && pwd )"
  ```
* Перебор целевых паттернов и рекурсивный поиск через `find`
* Функция `remove_item()`:

  * Удаляет элемент или выводит его путь в режиме dry-run

## Примечания

* Скрипт безопасно использовать в любом проекте Python, но внимательно проверяйте dry-run перед удалением.
* Можно расширять список `TARGETS` для удаления специфичных временных файлов проекта.
